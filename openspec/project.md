# Project Context

## Purpose
Top-AMS 是拓竹（Bambu Lab）打印机的第三方多色换色系统。项目通过 ESP32-C3 微控制器控制电机实现自动换色功能，追求更好的换色过程、更多的自定义配置以及更实惠的成本。

**核心原理**：
- 在拓竹换色 G-code 中插入热床温度改变并暂停
- 热床温度（1-17）对应不同的耗材通道
- ESP32 通过 MQTT 订阅得知热床温度变化
- 使用 MQTT 操作进退料并控制相应电机
- 最后恢复暂停继续打印

**支持范围**：
- 目前支持 PCB 设计最多 8 个通道，理论无通道数量上限
- 兼容机型：A1 mini、A1、P1S（固件版本 1.04.00 及以上）

## Tech Stack
- **编程语言**: C++20
- **开发框架**: ESP-IDF (>=4.1.0)
- **硬件平台**: ESP32-C3 微控制器（合宙开发板）
- **Arduino 兼容层**: espressif/arduino-esp32 (^3.1.3)
- **网络通信**: 
  - MQTT (MQTTS) - 与打印机通信
  - AsyncWebServer - Web 管理界面
  - WebSocket - 实时状态同步
- **JSON 处理**: bblanchon/arduinojson (^7.3.1)
- **构建系统**: CMake
- **Web 界面**: HTML/CSS/JavaScript（编译时转换为 C++ 头文件）

## Project Conventions

### Code Style
- **命名空间**: 使用命名空间组织模块（`config`、`mesp`、`mstd`、`esp`）
- **命名约定**:
  - 变量和函数使用小写字母和下划线（snake_case）
  - 结构体和类使用小写字母（如 `motor`、`SystemStateGuard`）
  - 常量使用中文或英文描述性名称
- **注释风格**: 
  - 支持中英文注释
  - 使用 `//` 单行注释和 `/* */` 多行注释
  - 关键功能使用 `@brief` 文档注释
- **代码组织**: 
  - 头文件使用 `#pragma once` 防止重复包含
  - 模块化设计，每个功能模块独立头文件
  - 用户配置区域明确标记（`//**********************用户配置区开始******************************`）

### Architecture Patterns
- **模块化设计**: 
  - `config.hpp` - 配置管理（电机配置、MQTT 配置等）
  - `channel.hpp` - 通道和锁机制
  - `espIO.hpp` - GPIO 和硬件接口
  - `espMQTT.hpp` - MQTT 通信
  - `bambu.hpp` - 打印机协议处理
  - `web_sync.hpp` - WebSocket 数据同步
  - `tools.hpp` - 工具函数
- **RAII 资源管理**: 
  - 使用 RAII 模式管理状态（如 `SystemStateGuard`）
  - 使用智能指针和 RAII 锁管理资源
- **异步任务处理**: 
  - 使用 FreeRTOS 任务处理并发操作
  - 使用信号量和通道进行任务间通信
- **配置驱动**: 
  - 通过编译时宏定义支持不同通道配置（MOTORS_4、MOTORS_6、MOTORS_8）
  - 运行时配置通过 Web 界面和持久化存储管理

### Testing Strategy
- **当前状态**: 项目主要依赖实际硬件测试和用户反馈
- **测试方法**: 
  - 硬件在环测试（HIL）
  - 实际打印机集成测试
  - Web 界面功能验证
- **验证点**: 
  - MQTT 连接和消息处理
  - 电机控制准确性
  - 换色流程完整性
  - Web 界面响应性

### Git Workflow

## Domain Context
### 3D 打印术语
- **AMS (Automatic Material System)**: 自动材料系统，拓竹打印机的多色打印系统
- **G-code**: 3D 打印机控制指令
- **热床温度**: 用于在 1-17 范围内传递通道信息（避免与正常打印温度冲突）
- **回抽 (Retraction)**: 打印头后退以停止挤出
- **冲刷体积 (Purge Volume)**: 换色时用于清理旧颜色的材料体积

### 硬件术语
- **GPIO**: 通用输入输出引脚
- **微动开关**: 用于检测进料位置的硬件开关
- **N20/TT 电机**: 不同型号的步进电机方案
- **通道**: 每个耗材颜色对应的独立进料通道

### 系统状态
- **操作状态**: `idle`（空闲）、`changing`（换色中）、`loading`（进料中）
- **系统锁定**: 防止在换色过程中进行其他操作
- **打印错误代码**: 
  - `0` - 正常
  - `260` (A1) / `259` (P1) - 退料完成需要退线
  - `262` - 进料检查
  - `263` - 进料冲刷
  - `768` - 进料完成

### 工作流程
1. **换色触发**: 打印机通过 G-code 设置热床温度（1-17）并暂停
2. **通道识别**: ESP32 通过 MQTT 订阅获取热床目标温度，映射到对应通道
3. **退料**: 控制当前通道电机退料，等待打印机完成退料流程
4. **进料**: 控制目标通道电机进料，等待打印机完成进料和冲刷
5. **恢复打印**: 解除暂停，继续打印

## Important Constraints
### 硬件约束
- **GPIO 限制**: 
  - GPIO 18、19 与 USB 冲突（Type-C 充电器可能影响）
  - GPIO 12、13 与 LED 灯冲突（使用通道 7 前需激活）
- **通道配置**: 支持 4、6、8 通道配置，通过编译时宏定义选择
- **内存限制**: ESP32-C3 内存有限，需注意内存使用

### 软件约束
- **固件版本兼容性**: 
  - 1.04.00 固件版本已验证支持
  - 1.05 固件版本增加了鉴权，当前可能不支持（需测试）
- **MQTT 连接**: 
  - 需要打印机局域网模式密码（开关状态不影响连接）
  - 需要设备序列号进行主题订阅
- **网络要求**: ESP32 和打印机需在同一局域网

### 功能约束
- **首层换色**: 默认设定为首层不会换色，需确认热端内通道为第一个颜色
- **料线准备**: 换色前需将待换料的料线推进至能在进料时间内进入热端的距离
- **温度范围**: 热床温度 1-17 范围用于通道信息传递，避免设置此范围内的热床温度

## External Dependencies
### 硬件依赖
- **主控模块**: 合宙 ESP32-C3 开发板

- **电机模块**: 
  - N20 电机方案（项目最早设计）
  - TT 电机方案（成本更低）
  - N20D 成对电机设计（适配 A1 龙门）

### 软件依赖
- **ESP-IDF**: Espressif 官方开发框架（>=4.1.0）
- **Arduino 兼容层**: espressif/arduino-esp32 (^3.1.3)
- **ArduinoJson**: JSON 序列化/反序列化库 (^7.3.1)
- **ESPAsyncWebServer**: 异步 Web 服务器 (^3.7.4)

### 外部服务
- **拓竹打印机**: 
  - MQTT 服务器（MQTTS，端口 8883）
  - 设备序列号用于主题订阅
  - 局域网模式密码用于认证
- **WiFi 网络**: ESP32 需要通过 SmartConfig 或 Web 界面配置 WiFi

### 开发工具
